1. Kết luận nhanh

Giai đoạn 1 Ổn khoảng 90–95%

Cần nhớ chuẩn hóa option trạng thái trong JSP admin.

Giai đoạn 2 Ổn khoảng 90–95%

Cần kiểm tra JDK/text block và hidden input trong các JSP.

Giai đoạn 3 Ổn khoảng 80–85%

Cần bổ sung thêm các điểm gọi cleanup và điều kiện an toàn.

SQL GĐ1–GĐ2 Đã xử lý constraint chính

Đã giải quyết lỗi CK_VE_trang_thai.

SQL GĐ3 Cần bổ sung

Cần thêm migration index và rà lại vài constraint/default.

Điểm quan trọng nhất: lỗi CK_VE_trang_thai đã được nhận diện đúng. Giải pháp thêm trạng thái Chờ thanh toán vào bảng VE là đúng. Tuy nhiên, sau Giai đoạn 3 nên có thêm migration riêng cho cleanup/index.

3. Rà soát Giai đoạn 2

Hướng thiết kế là đúng:

Chốt đơn:

Thanh toán thành công:
THANH_TOAN = Thành công

  • Không tin soTien từ client.

4. Rà soát Giai đoạn 3

Hướng tạo GiuChoCleanupService là đúng. Cleanup nên làm:

DAT_CHO Chờ thanh toán + hết hạn  → Hết hạn
GIU_CHO Đang giữ                 → Hết hạn
VE Chờ thanh toán                → Đã hủy
THANH_TOAN Pending               → Thất bại

Tìm chuyến
Chọn ghế
Checkout
Payment
Lịch sử đặt vé
Callback tới muộn

4.1. Bổ sung điều kiện không cleanup nhầm đơn đã thanh toán thành công

Trong GiuChoCleanupService, SQL tìm đơn quá hạn nên tránh hủy đơn đã có thanh toán thành công nhưng trạng thái DAT_CHO bị lệch.

Bản SQL an toàn hơn trong findExpiredDatChoIds(...):

StringBuilder sql = new StringBuilder();
sql.append(
        "SELECT dc.id "
      + "FROM dbo.DAT_CHO dc WITH (UPDLOCK, HOLDLOCK) "
      + "WHERE dc.trang_thai = ? "
      + "  AND dc.thoi_gian_het_han IS NOT NULL "
      + "  AND dc.thoi_gian_het_han < SYSDATETIME() "
      + "  AND NOT EXISTS ( "
      + "      SELECT 1 "
      + "      FROM dbo.THANH_TOAN tt "
      + "      WHERE tt.dat_cho_id = dc.id "
      + "        AND tt.trang_thai = N'Thành công' "
      + "  ) "
);

if (datChoId != null) {
    sql.append(" AND dc.id = ?");
}

4.2. Thiếu điểm gọi cleanup trong MyTicketServlet

Nếu người dùng truy cập thẳng /my-tickets?datChoId=..., có thể xem đơn/vé đã quá hạn mà chưa cleanup.

Cần thêm:

import vetau.service.GiuChoCleanupService;

private final GiuChoCleanupService cleanupService = new GiuChoCleanupService();

4.3. Thiếu điểm gọi cleanup trong TicketCheckService

Trang public tra cứu vé có thể tra ra vé Chờ thanh toán đã quá hạn nếu chưa có request nào khác kích hoạt cleanup.

package vetau.service;

import vetau.dao.TicketCheckDAO;
import vetau.dto.TicketCheckResultDTO;

import java.sql.SQLException;

public class TicketCheckService {
    private final TicketCheckDAO ticketCheckDAO = new TicketCheckDAO();
    private final GiuChoCleanupService cleanupService = new GiuChoCleanupService();

    public TicketCheckResultDTO traCuuVe(String maVe, String soGiayTo) throws SQLException {
        if (maVe == null || maVe.trim().isEmpty()) {
            throw new IllegalArgumentException("Vui lòng nhập mã vé.");
        }

        cleanupService.huyCacDonQuaHan();

        TicketCheckResultDTO result = ticketCheckDAO.findByMaVeAndSoGiayTo(maVe.trim(), soGiayTo);

        if (result == null) {
            return TicketCheckResultDTO.notFound("Không tìm thấy vé phù hợp. Hãy kiểm tra lại mã vé hoặc số giấy tờ.");
        }

        return result;
    }
}

5.2. SQL còn thiếu cho Giai đoạn 3

Nên tạo thêm file:

UPDATE_DB_GD3_CLEANUP.sql

Nội dung đề xuất:

USE VeTauDB;
GO

/* Index phục vụ tìm đơn quá hạn */
IF NOT EXISTS (
    SELECT 1
    FROM sys.indexes
    WHERE name = 'IX_DAT_CHO_Cleanup'
      AND object_id = OBJECT_ID('dbo.DAT_CHO')
)
BEGIN
    CREATE INDEX IX_DAT_CHO_Cleanup
    ON dbo.DAT_CHO (trang_thai, thoi_gian_het_han);
END
GO

/* Index phục vụ update giữ chỗ theo đơn */
IF NOT EXISTS (
    SELECT 1
    FROM sys.indexes
    WHERE name = 'IX_GIU_CHO_DatCho_TrangThai'
      AND object_id = OBJECT_ID('dbo.GIU_CHO')
)
BEGIN
    CREATE INDEX IX_GIU_CHO_DatCho_TrangThai
    ON dbo.GIU_CHO (dat_cho_id, trang_thai);
END
GO

/* Index phục vụ update vé theo đơn */
IF NOT EXISTS (
    SELECT 1
    FROM sys.indexes
    WHERE name = 'IX_VE_DatCho_TrangThai'
      AND object_id = OBJECT_ID('dbo.VE')
)
BEGIN
    CREATE INDEX IX_VE_DatCho_TrangThai
    ON dbo.VE (dat_cho_id, trang_thai);
END
GO

/* Index phục vụ update thanh toán theo đơn */
IF NOT EXISTS (
    SELECT 1
    FROM sys.indexes
    WHERE name = 'IX_THANH_TOAN_DatCho_TrangThai'
      AND object_id = OBJECT_ID('dbo.THANH_TOAN')
)
BEGIN
    CREATE INDEX IX_THANH_TOAN_DatCho_TrangThai
    ON dbo.THANH_TOAN (dat_cho_id, trang_thai);
END
GO

/* Index chống race khi kiểm tra ghế theo chuyến/ghế */
IF NOT EXISTS (
    SELECT 1
    FROM sys.indexes
    WHERE name = 'IX_VE_Chuyen_Ghe_TrangThai'
      AND object_id = OBJECT_ID('dbo.VE')
)
BEGIN
    CREATE INDEX IX_VE_Chuyen_Ghe_TrangThai
    ON dbo.VE (chuyen_tau_id, ghe_id, trang_thai);
END
GO

IF NOT EXISTS (
    SELECT 1
    FROM sys.indexes
    WHERE name = 'IX_GIU_CHO_Chuyen_Ghe_TrangThai_HetHan'
      AND object_id = OBJECT_ID('dbo.GIU_CHO')
)
BEGIN
    CREATE INDEX IX_GIU_CHO_Chuyen_Ghe_TrangThai_HetHan
    ON dbo.GIU_CHO (chuyen_tau_id, ghe_id, trang_thai, thoi_gian_het_han);
END
GO

6.1. Cập nhật GiuChoCleanupService.findExpiredDatChoIds(...)

private List<Integer> findExpiredDatChoIds(Connection conn, Integer datChoId) throws SQLException {
    StringBuilder sql = new StringBuilder();

    sql.append(
            "SELECT dc.id "
          + "FROM dbo.DAT_CHO dc WITH (UPDLOCK, HOLDLOCK) "
          + "WHERE dc.trang_thai = ? "
          + "  AND dc.thoi_gian_het_han IS NOT NULL "
          + "  AND dc.thoi_gian_het_han < SYSDATETIME() "
          + "  AND NOT EXISTS ( "
          + "      SELECT 1 "
          + "      FROM dbo.THANH_TOAN tt "
          + "      WHERE tt.dat_cho_id = dc.id "
          + "        AND tt.trang_thai = N'Thành công' "
          + "  ) "
    );

    if (datChoId != null) {
        sql.append(" AND dc.id = ?");
    }

    List<Integer> ids = new ArrayList<>();

    try (PreparedStatement ps = conn.prepareStatement(sql.toString())) {
        ps.setString(1, TrangThai.DAT_CHO_CHO_THANH_TOAN);

        if (datChoId != null) {
            ps.setInt(2, datChoId);
        }

        try (ResultSet rs = ps.executeQuery()) {
            while (rs.next()) {
                ids.add(rs.getInt("id"));
            }
        }
    }

    return ids;
}

8. Kết luận cuối cùng

Sau khi tự rà lại:

Giai đoạn 1: Có thể coi là ổn, chỉ cần nhớ chuẩn hóa option trạng thái trong JSP admin.
Giai đoạn 2: Thiết kế đúng, nhưng cần kiểm tra JDK/text block và hidden input ở booking-review/checkout.
Giai đoạn 3: Cần bổ sung cleanup ở MyTicketServlet, TicketCheckService, AdminBookingServlet, và thêm điều kiện không cleanup đơn đã có thanh toán Thành công.
SQL: GĐ1–GĐ2 đã xử lý lỗi constraint chính; GĐ3 cần thêm migration index và nên cập nhật lại file SQL tổng.
Chưa nên qua Giai đoạn 4 nếu chưa bổ sung các điểm còn thiếu của Giai đoạn 3.

Quan trọng nhất là 4 việc:

1. Sửa GiuChoCleanupService để không hủy đơn đã có THANH_TOAN Thành công.
2. Gọi cleanup thêm ở MyTicketServlet, TicketCheckService, AdminBookingServlet.
3. Chạy thêm SQL index Giai đoạn 3.
4. Rà lại text block nếu project dùng JDK 11.
Sau khi xong 4 điểm này, Giai đoạn 1–3 mới đủ chắc để chuyển sang Giai đoạn 4.
File HTML tổng hợp nội dung rà soát Giai đoạn 1–3 trước khi qua Giai đoạn 4.
